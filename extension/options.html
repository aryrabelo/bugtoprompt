BugToPrompt Options

Site repo bindings Map a site's hostname to a GitHub repo (owner/repo). BugToPrompt files issues there when capturing on that site. Host may be exact (app.example.com) or a wildcard (*.example.com).

Heads up: the sidecar only trusts exact origins (scheme + host + port) in its allowlist. For each non-localhost site, start the sidecar with BUGTOPROMPT_ALLOWED_ORIGINS=<scheme>://<host>[:<port>] — e.g. https://app.example.com or http://staging.example.com:8080 (comma-separate multiple) and restart it, or requests are blocked (CORS 403).
